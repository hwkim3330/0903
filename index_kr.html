📋 프로젝트 개요

본 프로젝트는 차량용 영역 기반(Zonal) 아키텍처에서 실시간 멀티미디어 스트리밍과 제어 데이터의 QoS를 보장하기 위한 TSN Credit-Based Shaper(CBS) 구현 및 성능 평가를 다룹니다. Microchip의 LAN9662 64포트 스위치와 VSC9959 TSN PHY를 기반으로 실제 차량 환경을 모사한 테스트베드를 구축하고, VLC 및 VOD 스트리밍 시나리오에서 CBS의 효과를 검증했습니다.

🔧 하드웨어 구성

Microchip LAN9662

  • 64개 기가비트 이더넷 포트
  • TSN 표준 지원 (802.1Qav, Qbv, AS)
  • 8개 트래픽 클래스
  • 하드웨어 CBS 가속
  • VelocityDriveSP SDK 지원
  • Cut-through switching
  • 9KB Jumbo frame 지원

VSC9959 TSN PHY

  • 5포트 기가비트 PHY
  • IEEE 802.1AS 시간 동기화
  • 하드웨어 타임스탬핑
  • Frame preemption (802.1Qbu)
  • Per-stream filtering (802.1CB)
  • SGMII/QSGMII 인터페이스

테스트 환경

  • 4K 카메라 시뮬레이터 (25Mbps)
  • FHD 카메라 x4 (각 8Mbps)
  • VLC 스트리밍 서버
  • VOD 서버 (nginx + RTMP)
  • iperf3 트래픽 생성기
  • Wireshark 패킷 분석기

💻 CBS 구현

VelocityDriveSP 기반 구현

Microchip VelocityDriveSP SDK를 활용하여 LAN9662의 하드웨어 CBS 기능을 구현했습니다. 각 포트별로 8개의 트래픽 클래스를 독립적으로 구성할 수 있으며, VLAN 태그 및 PCP 값을 기반으로 트래픽을 분류합니다.

/* LAN9662 CBS 구성 - VelocityDriveSP API */ #include <microchip/velocitydrivesp.h> // CBS 파라미터 설정 (포트 1, TC7 - 4K 비디오) vdsp_cbs_config_t cbs_config = { .port_id = 1, .traffic_class = 7, .idle_slope = 25000000, // 25 Mbps .send_slope = 975000000, // 975 Mbps (1G - 25M) .hi_credit = 3805, // (1522 * 25M) / 1G .lo_credit = 1483, // (1522 * 975M) / 1G .cbs_enable = true }; // CBS 적용 vdsp_cbs_port_config_set(device, &cbs_config); // VLAN to TC 매핑 vdsp_qos_vlan_tc_map_set(device, 100, 7); // VLAN 100 → TC7 vdsp_qos_vlan_tc_map_set(device, 110, 6); // VLAN 110 → TC6

🧪 실험 시나리오

시나리오 1: CBS 비활성화 (기준선)

모든 트래픽을 동일하게 처리하는 표준 이더넷 스위치 모드로 동작. 네트워크 혼잡 시 FIFO 큐잉으로 인한 패킷 손실과 지연 증가 예상.

  • 4K 스트림 1개 (25Mbps)
  • FHD 스트림 4개 (각 8Mbps)
  • VOD 스트림 8개 (각 4Mbps)
  • BE 트래픽 800Mbps

시나리오 2: CBS 활성화 (표준 예약)

각 트래픽 클래스별 대역폭 예약을 통한 QoS 보장. 실시간 스트림 우선 처리 및 BE 트래픽 제한.

  • TC7: 30Mbps 예약 (4K용)
  • TC6: 40Mbps 예약 (FHD용)
  • TC5: 35Mbps 예약 (VOD용)
  • 나머지: BE 트래픽

시나리오 3: 동적 부하 변화

실제 차량 환경을 모사한 동적 트래픽 패턴 테스트. 주행 상황별 네트워크 부하 변화에 대한 CBS 적응성 평가.

  • 0-30초: 경부하 (200Mbps)
  • 30-60초: 중부하 (500Mbps)
  • 60-90초: 과부하 (900Mbps)
  • 90-120초: 버스트 (1Gbps)
